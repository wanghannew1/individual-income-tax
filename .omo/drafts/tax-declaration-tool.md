# Draft: 个税申报数据整理工具（人力资源劳务派遣版）

## 项目定位
新建项目，替代原"个税累计预扣计算器"。

## 核心需求（已确认）
- 不计算累计应纳税所得额和个税
- 只整理当月收入、五险一金等扣除数据，生成国家个税端标准导入模板
- 支持员工库管理，优先导入税局人员信息表
- 自动带出零申报员工（本月无工资但在岗），可取消勾选
- 按甲方单位统计当月申报情况
- Web 界面操作
- 专项附加扣除由个人在个税 APP 端申报，企业端不报送
- 使用 SQLite 数据库存储员工、单位、历史申报记录
- 适配所有工资表格式：自动识别 + 手动校正列映射

## 技术决策
- **数据库**：SQLite（轻量本地，随项目迁移）
- **后端**：Python 3.12 + FastAPI
- **前端**：Jinja2 + 原生 HTML/CSS
- **ORM**：SQLModel 或 SQLAlchemy + Pydantic
- **Excel 解析**：pandas + openpyxl/xlrd
- **测试**：pytest
- **项目类型**：新建项目，清空原代码

## 业务模型
- 报税主体：人力资源公司（统一税号）
- 员工与人力资源公司签劳动合同，派驻到不同甲方单位
- 工资表按甲方单位划分，每月一个或多个 Excel 文件
- 甲方单位体现在申报表"备注"字段
- 零申报员工按最近一次工资表的甲方单位归属，如该单位已离职则不带出

## 关键数据实体
1. Employee（员工）
2. Unit（甲方单位）
3. MonthlyRecord（当月申报记录）
4. PayrollFile（上传的工资表文件）
5. ParserConfig（工资表列映射配置）

## 输出文件
1. 人员信息采集表（增员/更新）
2. 综合所得申报表（正常工资薪金所得）
3. 甲方单位汇总表

## 主要页面
1. 首页/仪表盘
2. 员工库管理
3. 月度申报流程
4. 历史记录查询

## 待实现优先级
Phase 1: 基础框架 + SQLite + 税局人员表导入 + 单格式工资表解析 + 申报表导出
Phase 2: 多文件上传 + 自动单位识别 + 通用列映射 + 零申报自动带出
Phase 3: 单位汇总 + 历史查询 + 配置持久化 + 完整测试

## 风险点
- 工资表格式差异大，需要灵活的列映射
- 单位名称不统一，需要别名处理
- 离职日期准确性影响零申报逻辑
- 税局模板字段可能调整
